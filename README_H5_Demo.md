# 年轮 H5 Demo 交付说明

> 构建版本：`h5-demo-20260713-v1`
> 交付物：`dist-h5-demo-20260713-v1.zip`（H5 静态资源包）
> 适用场景：Trae 创造大赛评委在普通电脑浏览器中 3 分钟内完整体验 A/B 端白金预彩排剧本

---

## 1. 启动方式

### 方式一：解压即用（推荐评委使用）

1. 解压 `dist-h5-demo-20260713-v1.zip` 到任意目录
2. 在解压后的目录中启动任意静态服务器：
   ```bash
   # 任选其一
   npx serve dist -l 3000
   npx http-server dist -p 3000
   python -m http.server 3000 -d dist
   ```
3. 浏览器打开 `http://localhost:3000`

### 方式二：直接打开 index.html

部分浏览器允许 `file://` 协议直接打开 `dist/index.html`，但为避免 CORS 限制导致 JS 模块加载失败，**强烈建议使用静态服务器方式**。

---

## 2. 环境要求

- 现代浏览器（Chrome / Edge / Safari 最新版）
- 屏幕宽度 > 500px 时以 375×812 手机壳居中呈现；< 500px 时全屏铺满
- 无需微信客户端、无需登录、无需云开发鉴权

---

## 3. 体验流程（3 分钟完整剧本）

### 第一步：环境选择
首次进入首页会弹出环境检测弹窗：
- **[我已在微信浏览器打开]**：走真实云函数动态联调（需微信客户端 + 已部署云函数）
- **[继续用非微信浏览器测试]**（评委推荐）：锁定白金预彩排剧本，固定输出顶级 LLM 回应

评委请点击第二个按钮。

### 第二步：A 端修复流程（林向阳视角）
1. 评审工具栏点击「切换到 A 端」
2. 首页点击「主动关心对方」进入修复 Wizard
3. Step 3 录入语料框已自动预填白金剧本前置词（月亮发烧 40 度事件）
4. 依次走完 Step 1→6：意图 → 情绪 → 语料 → 注入 → 渴望 → NVC 真心话
5. 点击「生成破冰微信小卡片」，A 端收到心晴安慰话

### 第三步：B 端回应流程（沈月亮视角）
1. 评审工具栏点击「切换到 B 端」→ 自动跳转到情绪防火墙
2. 选「还气着」→ 进入木偶替身宣泄，可宣泄 2 轮（每轮固定白金回复）
   - 或选「冷静了」→ 直接跳到结算页，心晴引导为正向肯定版本
3. 进入结算页看真心话，选「线下聊」或「线上聊」
   - **线上聊**：进入 chat_sandbox，B 端发送任意消息后 3 秒 A 端自动回复白金润色文案
   - **线下聊**：心晴给出祝福 + 见面 Tips

### 第四步：A 端和好攻略
1. 评审工具栏点击「切换到 A 端」回首页
2. 首页点击「Ta 给出回应啦，快看和好指南」
3. 查看白金 insight + 见面拥抱 Tips

### 评审工具栏（首页右下角悬浮）
- **切换到 A 端**：覆写 role 为 A，回到首页
- **切换到 B 端**：覆写 role 为 B，跳到防火墙
- **重置流程**：清空本地缓存（信箱/B 端选择/宣泄记录/聊天重置标记）
- **重新选择环境**：清空环境标记，重新弹出环境检测弹窗
- **模式指示器**：实时显示「白金剧本」或「动态云端」

## 5. 已知限制

1. **云函数不可用**：非微信浏览器环境下 `wx.cloud` 不可用，所有 LLM 调用走白金剧本固定文本，不依赖云函数
2. **chat_sandbox 实时同步**：H5 下 `db.collection().watch()` 不可用，改用本地 state 模拟，B 端发送后 3 秒预设 A 端回复
3. **双端合流**：评委需通过评审工具栏在单设备上手动切换 A/B 端视角，非真实双设备同步
4. **资源体积**：Taro + React 应用打包后部分 chunk 超 250KB，首次加载略慢，属正常现象
5. **微信浏览器分支**：若评委误点「微信浏览器」按钮，可点击评审工具栏「重新选择环境」回退

## 7. 重新构建

如需修改后重新出包：

```bash
npm run build:h5
Compress-Archive -Path dist\* -DestinationPath dist-h5-demo-20260713-v1.zip -Force
```

构建产物输出到 `dist/`，构建版本号在 `src/app.tsx` 的 `BUILD_VERSION` 常量中。
